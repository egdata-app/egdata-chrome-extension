# AGENTS.md

## Project

This is a Manifest V3 Chrome extension for egdata.app. It is built with Rsbuild, React 19, TanStack Router, TanStack Query, Tailwind CSS, and Biome.

## Commands

- Install dependencies: `pnpm install`
- Start development build: `pnpm dev`
- Typecheck: `pnpm typecheck`
- Lint/format check: `pnpm lint`
- Apply formatting: `pnpm format`
- Test: `pnpm test`
- Production build and zip: `pnpm build`
- Full verification: `pnpm check`

Use the local binaries in `node_modules/.bin` when pnpm's dependency-state checks interfere with one-off local validation.

## Architecture

- `src/background/` contains the MV3 service worker. Keep Epic cookie access, token handling, ownership validation, library sync, alarms, and external-message validation here.
- `src/content-script/` runs on `https://store.epicgames.com/*`. It should only ask the background service worker for ownership results; do not expose Epic tokens or write app data into the page origin.
- `src/main/` contains the extension UI at `main.html`. It uses TanStack Router with hash history because extension pages are served from a fixed HTML file.
- `src/lib/clients/` contains shared clients and message wrappers.
- `src/lib/services/` contains background-owned storage/sync/cache services.
- `public/manifest.json` is the production manifest. `public/manifest.dev.json` keeps local development-only permissions.

## Security And Privacy

- Never return Epic auth tokens through external messaging.
- Do not log cookie values, bearer tokens, or full auth payloads.
- Validate `sender.origin` for externally connectable messages.
- Keep production permissions narrow. Do not add `localhost`, broad `http://*/*`, `https://*/*`, or broad `web_accessible_resources` to the production manifest unless the user explicitly asks and the reason is documented.
- Prefer explicit user-driven login flows over opening hidden or pinned tabs.

## Coding Guidelines

- Keep TypeScript strict and avoid `any`.
- Prefer typed message contracts from `src/lib/messages.ts` for extension communication.
- Keep pure helpers side-effect free so they can be tested without Chrome APIs.
- Do not edit `src/main/routeTree.gen.ts` manually; it is generated by TanStack Router.
- Keep generated build output out of source changes unless the user specifically asks for a packaged artifact.

## Verification

Before handing off code changes, run:

```bash
pnpm typecheck
pnpm lint
pnpm test
pnpm build
```

For routing changes, verify extension URLs keep this shape:

- Home: `chrome-extension://<id>/main.html#/`
- Library: `chrome-extension://<id>/main.html#/library`
